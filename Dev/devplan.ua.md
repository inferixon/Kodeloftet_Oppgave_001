# План розробки портфоліо (навчальний проєкт)

Мета: зробити односторінкове портфоліо з центруванням контенту (рамка/контейнер), липким (sticky) хедером, чіткими секціями та адаптивністю від мобільних до 4K.

Не робимо (поки що): реальний/повний контент проєктів, складний JavaScript, важкі анімації.

---

## Інформаційна архітектура

Одна сторінка (один `index.html`):
- **Header (sticky)**: лого + горизонтальна навігація + соціальні іконки/посилання (GitHub, LinkedIn).
- **Main**:
	- **Hero**: великий заголовок “Velkommen”.
	- **About** (“Om meg”): текст + фото.
	- **Projects**: 3 картки в ряд (плейсхолдери) + простий hover-ефект.
	- **Background**: зображення + простий анімований ефект (parallax — опційно).
- **Footer**: горизонтальна панель із підписом + контактами.

---

## Правила дизайну / верстки

### Рамка сторінки (container)
- Зроби центрований контейнер для всього контенту (max-width), щоб на 2K/4K сайт не розтягувався занадто широко.
- Тримай однакові горизонтальні відступи.
- Параграфи — читабельна ширина (не “на весь екран”).

### Адаптивність
Брейкпоінти (просто):
- **Mobile**: 1 колонка, усе стеком. (опційно)
- **Tablet / small desktop**: 2 колонки там, де доречно. (опційно)
- **Desktop / 2K-4K**: 3 картки проєктів в один ряд, комфортні відступи.

Сітка проєктів:
- Mobile: 1 картка в ряд
- Tablet: 2 картки в ряд
- Desktop+: 3 картки в ряд

### Доступність (baseline)
- Усі пункти навігації — реальні `a` з `href="#..."`, і вони збігаються з `id` секцій.
- Є видимий focus-стан для клавіатури.
- Зображення мають змістовний `alt`.

## Покроковий план

### 0) Підготовка
- Визнач `id` секцій: `#hjem`, `#om`, `#prosjekter`, `#kontakt` (якщо додаємо).
- Визнач текст логотипу (ім’я або короткий бренд).
- Визнач соціальні лінки (мінімум GitHub; решта — потім).

Definition of done:
- Ти можеш описати структуру сторінки з пам’яті.

### 1) Header (sticky)
Що зробити:
- Хедер, який лишається зверху при скролі.
- Усередині 3 частини в один ряд:
	1) лого (зліва)
	2) навігація (по центру)
	3) соціальні іконки/лінки (справа)

Нотатки по верстці:
- Використай Flexbox для рядка хедера.
- Навігація — горизонтальний flex-список.

Definition of done:
- Хедер видно під час скролу.
- Лого, навігація і соц-лінки вирівняні акуратно.
- На малих екранах навігація може переноситись/стати в кілька рядків (без складних меню).

### 2) Hero
Що зробити:
- Hero-секція з сильним заголовком “Velkommen”.
- Короткий інтро-текст.

Definition of done:
- Чітка ієрархія: заголовок > текст.
- Hero нормально виглядає і на мобільному, і на великому екрані.

### 3) About (Om meg)
Що зробити:
- Секція з текстом + фото.

Нотатки по верстці:
- Mobile: фото під текстом або над текстом.
- Desktop: 2 колонки (текст + зображення).

Definition of done:
- Фото масштабується коректно (не вилазить за контейнер).
- Текст читабельний (не надто широкий).

### 4) Projects (3 картки + hover)
Що зробити:
- 3 плейсхолдер-картки проєктів.
- Кожна: заголовок, короткий текст, лінк/кнопка.

Ціль hover-ефекту (в стилі Steam, але просто):
- Наведення: легкий “підйом” + трохи тіні + ледь помітна зміна фону/scale.

Definition of done:
- Desktop: 3 картки в один ряд.
- Hover відчувається, але не заважає.
- Mobile: 1 картка в ряд.

### 5) Фон + простий анімований ефект
Два варіанти (обери один):
- **Варіант A (простий, бажано)**: фон-картинка на `body`/головному контейнері + дуже легка CSS-анімація (наприклад, повільний градієнт-оверлей).
- **Варіант B (parallax, потім)**: `background-attachment: fixed` тільки для десктопу + простий fallback для мобільного.

Definition of done:
- Фон не погіршує читабельність.
- Продуктивність не “вмирає” на мобільному.

### 6) Footer (flex-панель)
Що зробити:
- Футер як горизонтальний рядок з:
	- підписом/фразою
	- контактами (мінімум GitHub)

Definition of done:
- На десктопі футер вирівняний по горизонталі.
- На мобільному футер акуратно стає в стовпчик.

### 7) Фінальний адаптивний прохід (mobile → 4K)
Чекліст:
- Нема горизонтального скролу на жодній ширині.
- Контейнер центрований на широких екранах.
- Відступи виглядають нормально.
- Зображення різкі й адекватних розмірів.
